# 公开数据集

本项目不把大规模数据原文直接放进仓库，而是提供导入和转换脚本。原因：数据体积大、许可证可能变化、面试时更容易讲清楚"数据来自哪里、如何清洗、如何复现"。

## 主力数据源：JDDC-Baseline-Seq2Seq

本项目主力数据源为 [JDDC-Baseline-Seq2Seq](https://github.com/SimonJYang/JDDC-Baseline-Seq2Seq) 中包含的约 **1 万 session 脱敏电商客服对话**。这是京东 Dialog Challenge 大赛官方提供的基线数据，已做脱敏处理，可从 GitHub 直接下载。

**数据格式**（TSV）：
```
session_id    user_id    waiter_send    is_transfer    is_repeat    content
```
- `waiter_send=1`：客服消息
- `waiter_send=0`：用户消息
- 每条会话包含多轮对话，覆盖售前咨询、物流查询、退换货、投诉等真实场景

**导入方式**：

```bash
# 方式一（自动）：脚本会尝试从 GitHub 下载
uv run python scripts/import_jddc_baseline.py --sft-count 3000 --eval-count 500

# 方式二（手动）：如果自动下载失败
# 1. 浏览器打开 https://raw.githubusercontent.com/SimonJYang/JDDC-Baseline-Seq2Seq/master/data/chat.txt
# 2. 另存为 data/generated/jddc_chat.txt
# 3. 重新运行上面的命令
```

**脚本输出**：
- `data/generated/sft_train.jsonl` — SFT 格式训练数据（含场景分类）
- `data/generated/eval_set.jsonl` — 评测数据（含场景分类）

**注意**：raw.githubusercontent.com 在某些网络环境下可能不可达，建议通过浏览器手动下载。

## 其他可参考数据源

| 数据源 | 规模与特点 | 获取方式 | 备注 |
|--------|-----------|---------|------|
| JDDC（完整版） | >100 万对话，~2000 万 utterances | 需联系作者申请 | 更适合正式研究，个人不易获取 |
| JDDC 2.1 | ~24.6 万会话，300 万 utterances + 50.7 万图片 | 需填写申请表，机构邮箱发送 | 多模态电商客服，含商品知识库 |
| EcomGPT / EcomInstruct | ~250 万条电商指令数据，122 个任务 | ModelScope / GitHub | 适合补充商品理解和指令跟随能力 |
| Hugging Face 电商客服/QA 数据 | 格式差异大 | Hugging Face datasets | 适合快速 smoke test |
| 自构 SOP 样本 | 覆盖物流、退货、发票、投诉、安全拒答 | 自行构造 | 适合增强规则遵循和安全边界 |

## 面试里怎么讲

> 我不是直接拿来用，而是写了一个导入脚本，从 JDDC-Baseline-Seq2Seq 中提取会话数据，自动解析 QA 对并推断场景分类。这个脚本做得比较细致——它把多轮对话拆成单轮 QA，按 waiter_send 字段判断角色，还会对脱敏标记做正常化处理。另外还按关键词给每条对话打上场景标签，这样后面的 PGDM-DPO 才能按 policy 做加权采样。
>
> 没有被写死的数据路径。数据源可以无缝换成完整 JDDC 或自构 SOP 样本，脚本接口不变。
